When I [redesigned CSS Wizardry](https://csswizardry.com/2012/12/a-new-css-wizardry/) at the end of last year I ported it from WordPress to [Jekyll](https://github.com/mojombo/jekyll), and so far, so good.

One of the things I was very aware of, though, was that Jekyll had no way of supporting comments, where WordPress is built with commenting in mind. As such, I had to drop commenting from CSS Wizardry, with [a view to potentially reinstating it at a later date](https://github.com/csswizardry/csswizardry.github.com/issues/8).

1.  I wanted to be a little more selfish and experimental with the site, allowing me to try out a few new things that you can‚Äôt normally do in day-to-day client work.
